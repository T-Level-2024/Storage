Problem solving:

Problem solving in the T-Level Digital Design, Production and Development course is mainly about computational thinking, and how to approach problems.

Approaches to solving problems:

The three approaches to solving problems are top-down, bottom-up, and modularisation.

Top-down approach

Top-down approach is where problems are broken into smaller problems.

Bottom-up approach

Bottom-up approach is where we start with the smallest problem and we combine them with a smaller problem until eventually the whole problem is solved.

Modularisation approach

The modularisation approach is where problems are broken into components, and then assembled into a solution for the problem.


The four pillars of Computational Thinking:

These are the four pillars of Computational Thinking:
Decomposition
Pattern recognition
Abstraction
Algorithms

Decomposition

Decomposition is a method to understand a problem easier to solve it. It intends to break the problem up into smaller pieces, so that it is easier to understand.

Pattern recognition

After decomposition has been used, patterns can be more easily identified within the problems to make solving them easier.

Abstraction

Abstraction is where unnecessary data is removed to make it easier to understand.

Algorithms

Algorithms are a method where processes are repeated and expressions are used to help understand the problem easier and solve it.

Pseudocode

Pseudocode is code used to describe code colloquially and universally. It does not have any rules on what it is, and is generally used to explain code.

Program code

Program code is code that is written in a programming language. It is generally used to solve a problem or serve some purpose.

File handling:

File handling is an important part of Python, as it allows you to access data from files easily, and to use them as saves.


File handling is typically performed in this format:
file = open("myfile.txt", "r")

The second parameter in the open function is what is known as the text mode, or how the file is opened. It can be opened in many ways, here is a list of all text modes:

Text mode Name
"a" Append
"x" Create
"r" Read
"w" Write

There is also the option of choosing the file type to open. Python defaults to "t". Here is a list of file operators:

File operator Name
"t" Text file operator
"b" Binary file operator

Reading files

Files can be read using the .read() method, which reads one line:
line = file.read()

The whole file can also be read with the .readlines() method, which will read the entire file and output it as a list:
file = file.readlines()

Writing to files

Files can be written to using the .write() method, which writes one line:
file.write(line)
Multiple lines can also be written to the file using the .writelines() method, which will write the items of a list into the file:
file.writelines(next_lines_to_write)

Closing a file

Once you are finished with a file, and no longer need to interact with it, you should close the file. While you could leave it as it is, anything that was written to the file will not be saved:
file.close()

Creating a new file

Sometimes there isn't any file to begin with, so you'll need to create one. This can be done pretty easily using the "x" file operator:
file = open("mynonexistentfile.txt", "x")
